# 深度视频分析模板

当用户想要的不只是摘要，而是“像认真看过一样拆解”，优先使用这个结构。

## 1. 观看状态

- 是否真实播放。
- 是否听到口播。
- 是否有字幕、图表、屏幕录制或产品演示。
- 观看证据文件位置。

## 2. 时间线

按时间点写清楚：

- 画面发生什么。
- 口播/字幕说了什么。
- 重要转折、例子、对比或结论出现在哪里。

## 3. 博主到底在说什么

用 1-3 句话复述核心意思。不要照抄口播，要翻译成用户能直接理解的判断。

## 4. 论证拆解

- 主张是什么。
- 证据是什么。
- 推理链条是什么。
- 是否用了夸张、恐惧、机会焦虑、权威背书、案例包装或销售引导。

## 5. 可信度

把内容分成：

- 基本可信。
- 需要验证。
- 主要是观点或预测。
- 可能是营销包装。

## 6. 对用户的启发

结合用户正在做的事情：AI 工具链、视频分析、短视频生产、外贸独立站、skill、知识库、资产沉淀。

## 7. 用户应该做什么

给出具体行动，不要只写抽象建议。优先包括：

- 今天可以做的一个动作。
- 本周可以验证的一个实验。
- 可以沉淀到知识库的文档。
- 可以产品化、脚本化或 skill 化的机会。
